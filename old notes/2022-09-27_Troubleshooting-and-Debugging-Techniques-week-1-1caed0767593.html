Troubleshooting and Debugging Techniques weekÂ 1

troubleshooting: solving any kind of problem.
debugging: fixing bugs in the actual code.

tools: tcpdump(network), Wireshark(network).

Bash commands: (resources)ps, top, free. (calls)strace, ltrace.

problem solving steps(not always sequentially):
1. getting information: reproduction case.
2. finding the root cause.
3. performing the necessary remediation: short/long-term.
4. document what we do.

$ strace <app> #trace system calls
$ strace -o <.strace file> <app> #output into a file
$ less <.strace file> #displays the contents, one page at a time
$ <output> | less #same

getting information:
1. what were you trying to do?
2. what steps did you follow?
3. what was the expected result?
4. what was the actual result?

simplest explanations first

logs

/var/log/syslog #Linux system log(maybe outdated)
~/home/.xsession-errors #Linux user-specific log(maybe outdated)
$ jurnalctl #Linux
/Library/Logs #MacOs
eventvwr #Windows

https://manpages.ubuntu.com/manpages/kinetic/man1/journalctl.1.html

finding the root cause: do it in the test environment first.

$ iotop #current input/output operation
$ iostat #input/output operation
$ vmstat #virtual memory operation
$ iftop #current traffic on the network

intermittent issues: happened only occasionally.
1. modify the program to get more information.
2. debugging mode that generates more information.
3. monitoring the environment

Heisenbugs: bug with observer effect.

binary search: sort the list and search middle, can eliminate half of the list.

$ git bisect <commit1> <commit2>
