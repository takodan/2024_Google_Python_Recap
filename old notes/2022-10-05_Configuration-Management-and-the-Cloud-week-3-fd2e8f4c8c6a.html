Configuration Management and the Cloud week 3

cloud services types:
1. software as a service (SaaS): deliver an entire app to the customer. the cloud provider manages everything.
2. platform as a service (PaaS): offers a preconfigured platform to the customer. like an SQL database as a service.
3. infrastructure as a service(IaaS): supplies only the bare-bones computing experience. let you run virtual machines in the cloud.

consider databases regions for latency and policy reasons.

queries per second (QPS)

it is easier to scale in the cloud rather than on-premise.

horizontal scaling: add or remove servers.
vertically scaling: make servers bigger or smaller.

containers: apps that are packaged together with their configuration and dependencies.

public, private, hybrid, and multi-cloud.

reference images: store the contents of a machine in a reusable format. if it is a snapshot of a disk, call disk image.
templating: capturing all of the system configurations to let us create VMs in a repeatable way.

etc/systemd/system/ is the default systemd directory in many Linux distros.

managing instances in the Cloud:
1. create a VM in IaaS (in the web wizard).
2. set up the VM (in the terminal, connect to VM).
3. create an image of the VM (in the web wizard).
4. use the image to create an instance template (in the web wizard).
5. create a new VM for the template (in the web wizard).
6. or using the command provided by the service in the terminal.

more detail on creating a VM in google cloud:
https://cloud.google.com/compute/docs/quickstart-linux
https://cloud.google.com/compute/docs/instances/create-vm-from-instance-template
https://cloud.google.com/sdk/docs

load balancer: balance requests.

Round robin: a strategy load balancer use, give each node one request.

autoscaling: when scaling, some nodes will be considered ephemeral. content to a database to get the data if data is missing.

content to a site:
1. retrieve an IP address that identifies the entry point for the sites. a single website could have several entry points.
2. web cache with a load balancer (Varnish cache or service like Cloudflare and Fastly).
3. if the info being requested is not in caches, go to the web service with a load balancer to get the actual web services.
4. web services may connect to the database and/or DB cache (Memcached and Redis).

orchestration: the automated configuration and coordination of complex IT systems and services, by making infrastructure as code (Terraform or other tools provided by cloud vendors). normally also need to use an API to do this.
hybrid cloud: only part of services are in the cloud.
