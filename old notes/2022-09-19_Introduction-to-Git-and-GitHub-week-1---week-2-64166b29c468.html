diff and patch

$ diff <file1> <file2>  #get different line
$ diff -u <file1> <file2> #get unified format of differences
$ diff -u <file1> <file2> > change.diff #create diff file
$ patch <file> < <diff_file> #modified with diff file

VCS: version control system.
commit: a collection of edits.
repository: contains software projects or code

git

modified > staged > committed
staging area: all of the information is going to go into the next commit.

$ git config -l #show current configuration
$ git config --global user.name "<name>" 
#<--global> set for all repositories
$ git config --global user.email "<email>"
$ git init #initialized git repository in current directory
$ git clone #copy repository
$ git add <file> #add file to staging area
$ git status #show current working tree and pending changes
$ git commit #open a commit message and commit it
$ git commit -m "<commit_message>"
$ git log

good commit message: short summary > blank line > details. make sure a line is not too long.

skipping the stage area:

$ git commit -a # stage any changes to tracked files and commit them

git uses the HEAD alias to represent the currently checked-out snapshot.

more information about changes:

$ git log -p -2
#listing last 2 commit diff -u, listing all without <-2>
$ git show <commit_ID> 
#only show the ID commit diff -u, you can use only 4~8 ID characters
$ git log --stat #commit file and line added or remove
$ git diff #show unstage changes
$ git add -p #show unstage changes and ask next move
$ git diff --staged #show staged but not commit

delete, rename, and ignore:

$ git rm <file> #stop trackd, remove file. commit is needed
$ gir mv <old_name> <new_name> #move or rename. commit is needed
$ echo <file_want_to_ignore> > .gitignore
#use <ls -la> show all file include .file
#then you can see .gitignore file it containing a list of files for
#git to skip, need to add and commit .gitingore
$ git add .gitignore
$ git commit -m "<string>"

undo and rollback:

$ git checkout <file> #reverts changes before they are staged
#<-p> for specific change
$ git reset HEAD <file> # remove file from staging area
#<-p> for specific change
$ git commit --amend 
#take staging area update and overwrite the previous commit
#avoid amandong public commits
$ git revert HEAD
#creat reverts commit,from HEAD, history remains
$ git revert <commit_ID>
#creat reverts commit,from ID commit, history remains

branch and merging:

branch: a pointer to a particular commit.
master: a default branch when a new repository is initialized.
merging: combining branched data and history together.

$ git branch #show existing branch
$ git branch <branch_name> #create new branch
$ git checkout <branch_name> #switch to branch_name branch
$ git checkout -b <branch_name> #creat new branch and switch to it
$ git branch -d <branch_name> #delete branch_name branch
#if there are changes have not been merged back into master branch
#it shows an error. use <-D> to delete a not merged branch
$ git merge <branch_name> #merge to current branch(HEAD), usually use master(<git checkout master>)

fast-forward merge: if the branch you are merging is after the master branch, it will simply move the pointer to point to the same commit.

merge conflicts: when merging to the same part of the same file.

#when merge failed because of conflicts 
$ git status #to get more information
$<(edit the file)>
#it will add some infromation about conflicts line
#fix the conflicts
$ git add <fixed_file>
$ git status #to see they are fixed or not
$ git commit
$ git merge --abort #abort mission
$ git log --graph --oneline
#show commit log graphically, one line per commit
