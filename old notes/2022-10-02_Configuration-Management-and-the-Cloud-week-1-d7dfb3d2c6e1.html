Configuration Management and the Cloud week 1

scale: keep achieving larger impacts with the same amount of effect. a scalable system is a flexible system.

configuration management system: Puppet, Chef, ansible, and CFEngine.

infrastructure as code(IaC): configuration in infrastructure is stored in a version control system. it is consistent, versioned, reliable, and repeatable.

cron job: schedule tasks in Linux.

Puppet resources: the basic unit for modeling configuration.

domain-specific language(DSL): a programming language that is more limited in scope.
https://en.wikipedia.org/wiki/Domain-specific_language

#basic syntax
clsaa <name> {
  <resources_type> { '<title>':
    <attributes> => <value>,
  }
}
#example
clsaa ntp {
  #make sure ntp package is latest
  package {'ntp':
    ensure => latest,
  }
  #set ntp.conf file with source file
  file {'/etc/ntp.conf':
    source => 'puppet:///modules/ntp/ntp.conf',
    replace => true,
  }
  #make sure ntp service is enabled and running
  service {'ntp':
    enable => true,
    ensure => running,
  }
}

declarative language: declare the state that we want to achieve rather than the procedure computer needs to follow to reach the gold.
idempotent action: can be performed repeatedly without changing the system after the first run.
“test and repair” paradigm: actions are taken only when they are necessary.
Puppet is stateless: no state is being kept between runs.
